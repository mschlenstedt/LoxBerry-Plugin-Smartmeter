# SmartMeter v2 User Guide

## Overview

SmartMeter v2 reads smart meter data on LoxBerry. The legacy mode supports optical I/R reading heads connected below `/dev/serial/smartmeter/`.

The plugin can publish the values it reads through several outputs:

- HTTP: values can be read from the plugin web frontend.
- UDP: values are sent to all configured Miniservers.
- MQTT: values are published through the LoxBerry MQTT Gateway.

The MQTT and UDP outputs use the same value set.

## Requirements

- LoxBerry with the SmartMeter v2 plugin installed.
- At least one supported optical I/R reading head.
- A configured meter in the legacy SmartMeter v2 configuration.
- For MQTT publishing: LoxBerry MQTT Gateway must be installed and configured.

## Configuration

Open the SmartMeter v2 plugin in the LoxBerry web interface and switch to the legacy configuration.

### Automatic Reading

Enable periodic reading if the plugin should read meters automatically.

Available intervals include:

- minimal
- every minute
- every 3, 5, 10, 15, 30, or 60 minutes

You can also trigger a manual meter read from the plugin web interface.

### UDP Output

Enable **Send data by UDP** to send the meter values to the configured Miniservers.

The UDP port can be configured in the plugin settings. The data is sent to all configured Miniservers.

### MQTT Output

Enable **Send data by MQTT** to publish meter values through the LoxBerry MQTT Gateway.

The plugin uses the LoxBerry MQTT Gateway UDP input port from `general.json`. It does not require separate broker credentials in the plugin settings.

Configure the MQTT base topic in the plugin settings.

Default:

```text
smartmeter
```

Topic structure:

```text
<base topic>/<meter>/<value name>
```

Example:

```text
smartmeter/ABC123/Consumption_Total_OBIS_1.8.0
```

The MQTT payload is the value only.

Example payload:

```text
12345.678
```

Messages are published with the retain flag.

## Meter Values

The value names are generated by the meter reader. Typical examples are:

- `Last_Update`
- `Last_UpdateLoxEpoche`
- `Consumption_Total_OBIS_1.8.0`
- `Consumption_Power_OBIS_1.7.0`
- `Delivery_Total_OBIS_2.8.0`
- `Total_Power_OBIS_15.7.0`

The available values depend on the meter type and protocol.

## Troubleshooting

### UDP works, but MQTT values do not appear

Check the following:

- MQTT output is enabled in the plugin settings.
- The MQTT base topic is not empty and does not contain MQTT wildcards `#` or `+`.
- LoxBerry MQTT Gateway is installed and running.
- The MQTT Gateway UDP input port is configured in LoxBerry.
- Subscribe to `<base topic>/#` in the MQTT Gateway or with an MQTT client.

### No meter data is available

Check the following:

- The I/R reading head is connected.
- The device exists below `/dev/serial/smartmeter/`.
- The meter configuration is complete.
- Manual reading from the plugin web interface works.

### Log Files

The plugin writes runtime logs below the LoxBerry plugin log directory. In the legacy frontend, use the log view to inspect meter read and publish activity.
